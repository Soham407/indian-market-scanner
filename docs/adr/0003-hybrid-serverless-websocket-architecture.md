# Hybrid Architecture: Pure Supabase for Paper Trading, +VPS Worker for Real Money

**Status:** Accepted (supersedes ADR-0001 for the auto paper-trading bot scope)
**Date:** 2026-05-21

The auto paper-trading bot uses two architectures across its lifecycle. In v1 (months 1–3, paper trading only), the entire bot runs on Supabase Edge Functions with cron-based scheduling — paper trades use 1-minute candle OHLC to detect stop/target hits, which is industry-standard for simulated execution and removes the need for any persistent process or real-time tick feed. In v1.2 onward (real money), a small Node WebSocket worker is added on a DigitalOcean Bangalore droplet ($5/mo, covered by GitHub Student Pack credits) to maintain a persistent low-latency connection to Angel One SmartAPI for sub-second stop and target execution on real orders; Supabase remains the database and the source of truth, and the droplet is purely the execution arm. Pure-Supabase real-money trading was rejected because 1-minute cron granularity systematically misses fast moves and defeats the daily drawdown circuit-breaker. A US-region serverless platform (Railway/Render) was rejected for real money because the 200–400ms latency to Indian markets adds meaningful slippage on every stop fill — at retail scale, ₹400/month for a Bangalore droplet recovers its cost within days of avoided slippage. ADR-0001 still applies to the old "Market Sniper" dashboard scope but is superseded for this bot.
